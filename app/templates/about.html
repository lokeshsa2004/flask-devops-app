About Flask DevOps App

A complete guide to deploying Flask applications on production servers

🎯 Mission

This project demonstrates a production-ready Flask application with:

  • Scalable Architecture: Blueprint-based modular design
  • Database Integration: SQLAlchemy ORM with MySQL
  • RESTful APIs: JSON endpoints for frontend integration
  • Modern Frontend: Dynamic JavaScript with form handling
  • DevOps Ready: Docker, Docker Compose, systemd
  • CI/CD Pipeline: GitHub Actions automation

📚 Technology Stack

Backend

  • Python 3.11 - Modern Python runtime
  • Flask 2.3+ - Lightweight web framework
  • SQLAlchemy - ORM for database operations
  • Flask-Migrate - Database migrations

Database

  • MySQL 8.0 - Relational database
  • PyMySQL - Python MySQL connector
  • Connection Pooling - For performance

Server Stack

  • Gunicorn - Production WSGI server
  • Nginx - Reverse proxy and static files
  • Docker - Containerization
  • systemd - Process management

DevOps & CI/CD

  • GitHub Actions - Automated workflows
  • GHCR - Container registry
  • AWS EC2 - Cloud deployment target
  • Docker Compose - Local orchestration

🏗️ Architecture Overview

Internet

Users access via public IP or domain

↓
Nginx (Port 80/443)

Reverse proxy, static file serving, SSL termination

↓
Gunicorn (Port 8000)

WSGI server with multiple worker processes

↓
Flask Application

Blueprints, routes, business logic

↓
SQLAlchemy + MySQL

Data persistence and ORM layer

🚀 Deployment Options

🖥️ Bare Metal / VM

systemd service + Nginx on Ubuntu

Best for: Full server control, on-premise

🐳 Docker Container

Single container with Gunicorn

Best for: Quick testing, simple deployments

🔗 Docker Compose

Nginx + Flask + MySQL stack

Best for: Local development, multi-service apps

☁️ AWS EC2 + GitHub Actions

Automated CI/CD with Docker deployment

Best for: Production, continuous delivery

📖 Getting Started

Local Development

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
gunicorn -w 3 -b 127.0.0.1:8000 app:app

Docker Compose

export APP_IMAGE=flask-devops-app:local
docker compose up -d
# Open http://localhost/

Production on EC2

git clone <repo> flask_app
cd flask_app
# Configure secrets in GitHub
git push origin master
# Automated deployment via CI/CD
