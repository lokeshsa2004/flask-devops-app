## ⚙️ GITHUB ACTIONS - REQUIRED SECRETS

### 4 Secrets You Must Add

Go to: **GitHub Settings → Secrets and variables → Actions → New repository secret**

```
EC2_HOST = Your EC2 IP address (e.g., 203.0.113.42)
EC2_USER = ubuntu
EC2_KEY  = Contents of your .pem file (full key)
GHCR_TOKEN = GitHub Personal Access Token
```

### How to Generate EC2_KEY

**If you have .pem file:**
```powershell
# Windows PowerShell
$keyContent = Get-Content "C:\path\to\your-key.pem" -Raw
$keyContent | Set-Clipboard
# Paste in GitHub secret
```

### How to Generate GHCR_TOKEN

1. Go to https://github.com/settings/tokens
2. Click "Generate new token (classic)"
3. Select scopes: `write:packages`, `read:packages`, `delete:packages`
4. Copy token and paste in GitHub secret

## 🆘 QUICK TROUBLESHOOTING

### "Port 80 already in use"
```bash
# Change port in docker-compose.yml
# Or find and stop process using port 80
lsof -i :80
kill -9 <PID>
```

### "Docker daemon not running"
```bash
# Start Docker
systemctl start docker  # Linux
# or use Docker Desktop GUI
```

### "Tests fail"
```bash
pip install -r requirements.txt pytest pytest-cov
pytest tests/test_app.py -v --tb=short
```

### "GitHub Actions deploy fails"
```
1. Verify all 4 secrets are set
2. Check EC2 instance is running
3. Verify EC2 security group allows SSH (port 22)
4. Test SSH connection manually first
```

**See [ANALYSIS_SUMMARY.md](./ANALYSIS_SUMMARY.md#-troubleshooting) for complete guide**
